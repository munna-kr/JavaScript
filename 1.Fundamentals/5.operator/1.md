## JavaScript Operator Precedence Notes  

## **1. What Is Operator Precedence?**

Operator precedence determines which operations run first in an expression.

**Higher precedence = evaluated earlier.**

### **Example**
```js
2 + 3 * 4
// 3 * 4 runs first → 12
// Result = 2 + 12 = 14
```

## **2. Full Precedence Table (Highest → Lowest)**
 21. Parentheses → (...)
 20. Member Access & Calls → obj.x, arr[i], fn(), new.target
 19. new (no parentheses) → new Foo
 18. Postfix & Unary → x++, ++x, !x, typeof, await
 16. Exponentiation → **
 15. Multiplicative → *, /, %
 14. Additive → +, -
 13. Bitwise Shifts → <<, >>, >>>
 12. Relational → <, >, <=, >=, instanceof, in
 11. Equality → ==, !=, ===, !==
 10. Bitwise AND → &
  9. Bitwise XOR → ^
  8. Bitwise OR → |
  7. Logical AND → &&
  6. Logical OR → ||
  5. Nullish Coalescing → ??
  4. Ternary → ? :
  3. Assignment → =, +=, -=, **=, etc.
  1. Comma → ,

## **3. Associativity**

Defines which direction operators evaluate when precedence is the same.

### **Left → Right**
```js
+, -, *, /, %, <, >, ==, ===, &&, ||
```

### **Right → Left**
```js
=, +=, -=, **, unary (!, typeof), ternary ?:
```

## **4. Examples**

### **Example 1 — Parentheses vs Multiplication**
```js
10 + 2 * 3
// → 16

(10 + 2) * 3
// → 36
```

### **Example 2 — Logical AND vs OR**
```js
false || true && false
// true && false = false
// false || false = false
```

### **Example 3 — Nullish (??) vs OR (||)**
```js
0 || 5  // → 5 (0 is falsy)
0 ?? 5  // → 0 (0 is NOT null/undefined)
```

### **Example 4 — Assignment Right-to-Left**
```js
let a, b;
a = b = 10;
// b = 10 happens first
// a = 10
```

### **Example 5 — Ternary + Logical**
```js
true && false ? "yes" : "no"
// true && false = false
// false ? "yes" : "no"  → "no"
```

### **Example 6 — Mixing || and ??**
```js
let x = null;
x ?? false || true
// || has higher precedence than ??
// x ?? (false || true)
// null ?? true = true
```

## **5. Memory Tricks**

**1. Easy order to remember**
# Parentheses → Member → Unary → Math → Bitwise → Logical → Ternary → Assignment → Comma

**2. "Math first, logic last"**
# Math operations execute early; logical operators come late.

**3. Assignment runs near the bottom**
# And it evaluates **right → left**.

**4. Exponentiation is right-associative**
# `2 ** 3 ** 2` → `2 ** (3 ** 2)`.

**5. Always use parentheses when mixing `||`, `&&`, `??`**
# Their precedence can cause surprising results.
