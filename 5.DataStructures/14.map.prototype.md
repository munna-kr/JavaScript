# Map.prototype

-> it is the object that holds all the shared methods of every Map instance.
-> All Map instances inherit their behavior from Map.prototype.

> behind the scenes
m (instance)
↓
Map.prototype (where set(), get(), has(), etc. live)
↓
Object.prototype
↓
null

>
Map.prototype
├─ constructor (Map)                         ← points back to Map constructor
├─ clear()                                   ← remove all entries
├─ delete(key)                               ← remove one entry
├─ get(key)                                  ← read value for key
├─ has(key)                                  ← boolean if key exists
├─ set(key, value)                           ← add or update entry
├─ keys() → Iterator<key>                    ← iterate keys (in insertion order)
├─ values() → Iterator<value>                ← iterate values (in insertion order)
├─ entries() → Iterator<[key, value]>        ← iterate [key, value] pairs (in insertion order)
├─ [@@iterator] (same as entries)             ← `for (const [k,v] of map)` uses this
├─ forEach(callback, thisArg?)                ← call callback for every entry
├─ get size (accessor property)               ← number of entries (getter, not a method)
├─ @@toStringTag (Symbol.toStringTag)         ← "Map" (affects Object.prototype.toString)
└─ (other well-known symbol properties may exist in some engines)
