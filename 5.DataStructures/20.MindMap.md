--> In JavaScript, Map, Set, Array, and Object are Built-in constructor functions.
--> technically functions that create and initialize objects using new

--> these constructors just create instances that store only their data.

--> Methods are not stored in the instance
-->
All behavior (methods) is stored in 
  .prototype

Map.prototype

Set.prototype

Array.prototype

Object.prototype

--> all methods are shared through their prototype objects using the prototype chain.

JavaScript Built-ins (Constructors)
├── Object()
│   ├── Instance stores: key-value data
│   ├── Methods in: Object.prototype
│   └── Prototype chain:
│       instance → Object.prototype → null
│
├── Array()
│   ├── Instance stores: indexed elements + length
│   ├── Methods in: Array.prototype (push, map, etc.)
│   └── Prototype chain:
│       instance → Array.prototype → Object.prototype → null
│
├── Map()
│   ├── Instance stores: [[MapData]] key-value entries (internal)
│   ├── Methods in: Map.prototype (set, get, has, etc.)
│   └── Prototype chain:
│       instance → Map.prototype → Object.prototype → null
│
└── Set()
    ├── Instance stores: [[SetData]] unique values (internal)
    ├── Methods in: Set.prototype (add, has, delete, etc.)
    └── Prototype chain:
        instance → Set.prototype → Object.prototype → null
